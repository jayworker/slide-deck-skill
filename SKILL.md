# Slide Deck — Traction Dashboard 발표덱 제작

연구실 발표용 HTML 슬라이드 덱을 만드는 스킬. 산출물은 **단일 HTML 파일** (16:9 슬라이드 세로 스택, 브라우저 F11로 발표).

## 워크플로

1. **디자인 소스 확인**: 작업 폴더에 `design.md`가 있으면 그 토큰을 우선 사용. 없으면 이 스킬의 `template.html`에 내장된 Traction 토큰 사용.
2. **슬라이드 목록 설계**: 덱 구조는 `표지(큰 그림) → 개념/기초 → 본문/예시 → 액션 아이템`. 슬라이드당 주장 1개 — 주장이 두 개면 슬라이드를 쪼갠다. 러프한 제안 단계면 5~9장으로 제한.
3. **`template.html`을 베이스로 작성**: 이 스킬 폴더의 `template.html`을 Read해서 CSS 토큰·슬라이드 클래스·자동 번호 스크립트를 그대로 가져오고, 슬라이드 섹션만 교체한다. 산출 파일명은 `<주제>_deck.html`, 작업 폴더에 저장.
4. **완성 후**: `Start-Process "<파일경로>"`로 브라우저에서 열어준다. 사용자가 스크린샷으로 잘림/뭉개짐을 보고하면 → 아래 "오버플로 대응" 절차.

## 슬라이드 패턴 (모든 본문 슬라이드 공통 3층 구조)

```
[배지 + 제목 + .pg 페이지번호]
[statement — 큰 한 줄 메시지]   ← 이 슬라이드의 유일한 주장
[히어로 비주얼 — SVG 또는 표]    ← 주장을 그림이 증명
[키워드 칩 × 3]                 ← 기억할 것 3개
```

- **표지**: 배지 + 큰 제목(핵심어 그린) + 한 줄 부제 + 하단 단계/KPI 카드 행(`→` 연결)
- **개념**: statement + 히어로 + 칩 3개, 배지 `Basics n/m`
- **예시/데이터**: statement(질문형) + 좌 변수카드(라벨+키워드 나열) + 우 미니 표 + 한 줄 노트
- **액션**: 큰 번호 카드 3개 — 번호 + 명령형 제목 + 한 줄 + 담당/기한(mono)

## 글쓰기 규칙 (가장 중요 — 사용자 확정 피드백)

- **문장 단락 금지.** 카드 텍스트 = 키워드 + 최대 1줄 캡션. "측정 조건을 반드시 기록해야 한다" → "측정 조건? 단위?"
- 숫자·ID·변수명은 JetBrains Mono: `r = +0.71`, `LMR-001`
- 이모지·느낌표·들뜬 카피 금지. 절제된 학술 톤.

## 색 사용 규칙 — 색 하나 = 의미 하나

한 슬라이드 안에서 같은 색이 두 가지 의미(강조 + 데이터 값)를 지면 독자가 구분하지 못한다. 강조와 데이터의 색 채널을 분리한다.

### 강조 (텍스트)

| 색 | 용도 |
|----|------|
| 그린 `#0E9F6E` (.green) | 핵심 개념·긍정·허용·최적 — 슬라이드당 1~3곳만, **볼드 또는 큰 글자에만** (배경 대비 3.3:1이라 소형 일반 텍스트엔 부족) |
| 잉크 볼드 `#111827` | 보조 강조 (그린과 짝: 입력 X는 그린, 출력 Y는 잉크 볼드) |
| 레드 `#D64545` (.red) | 금지·불가·✗ **전용**. 다른 용도 금지 — 데이터 시리즈·음수에도 쓰지 않는다 |

### 데이터 (차트·히트맵·표) — template의 `--cat/--seq/--div` 토큰 사용

- **크기(magnitude)** = 시퀀셜: 그린 한 색의 밝기 램프 `--seq-1..4` (연→진)
- **부호(polarity: 상관계수, 증감)** = 다이버징: 음수 `--div-neg`(블루) · 중립 `--div-mid`(회색) · 양수 `--div-pos`(그린). **음수를 회색조로 칠하지 말 것** — 부호가 크기처럼 읽힌다. 레드는 여기서도 금지 전용이라 침범 불가.
- **시리즈(identity)** = 카테고리 `--cat-1..4` **고정 순서**(그린→블루→앰버→퍼플), 건너뛰기·재배열 금지 (순서 자체가 색약 안전장치). 5개째부터는 회색 "기타"로 묶는다. 단일 시리즈면 무조건 그린.
- **색+형태 이중 인코딩**: 색만으로 의미를 싣지 않는다 — 점선=금지 / 실선=허용, 히트맵 셀엔 숫자 병기, 범례·라벨 텍스트는 잉크/라벨색(시리즈 색으로 텍스트를 칠하지 않는다 — 색은 옆의 마크가 진다).

## 비주얼 규칙

- 모든 그림은 **인라인 SVG로 직접 그린다** (산점도, 히트맵, 곡선, 분기 다이어그램). 외부 이미지·아이콘 폰트 금지. 원형 차트 금지.
- **아이콘은 Lucide 단일 소스** (lucide.dev, ISC). 이 스킬 폴더의 `icons/*.svg`에 자주 쓰는 것들이 동봉되어 있다 — Read해서 **인라인 SVG로 삽입** (경로 데이터를 손으로 그리지 말 것). 없는 아이콘은 `https://unpkg.com/lucide-static/icons/<이름>.svg`에서 가져온다. stroke 1.75 · `currentColor` · fill none 유지, `class="ico"`(1.15em)로 크기 통일. 색은 텍스트와 같은 규칙(핵심만 그린, 금지는 레드). 아이콘은 키워드를 보조할 때만 — 장식용 나열 금지.
- **금지/허용 대비 = blocked-path 패턴**: 빨간 점선 화살표가 굵은 ✗에 막혀 끊기고(도달 실패), 초록 실선 화살표는 화살촉과 함께 목적지 도달. template.html에 예시 있음.
- 데이터 표가 히어로가 될 수 있다 — X 구간 헤더는 그린 2px 밑줄, Y 구간은 잉크 2px 밑줄로 밴드 구분.
- 수식 필요 시 KaTeX CDN + auto-render(`$$...$$`). template.html 주석 참조.

## 엔지니어링 규칙

- **페이지 번호 하드코딩 금지** — template의 JS가 `.pg`와 footer를 자동 채움. 슬라이드 추가/삭제/이동 자유.
- **프레젠테이션 모드 내장** — template의 내비게이션 JS: →/←, PgDn/PgUp, Space, Home/End로 슬라이드 단위 이동(스크롤 스냅). 사용자에게 "F11 전체화면 + 방향키"로 발표하라고 안내할 것.
- 16:9 고정, 슬라이드 내부는 `font-size: clamp()` 기반 em 스케일.
- 폰트/KaTeX는 CDN — 오프라인 발표 가능성 있으면 사용자에게 알릴 것.

## 오버플로 대응 (스크린샷 피드백 시)

고정 높이 카드에서 텍스트 잘림 발생 시 순서대로:
① 텍스트 칼럼 폭 확대(그리드 비율 조정) → ② 폰트 한 단계 축소 + 행간 축소 → ③ `overflow:hidden` 안전장치 → ④ 그래도 넘치면 텍스트를 줄이거나 슬라이드 분할.
